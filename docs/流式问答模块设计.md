# 流式问答模块设计

> 对齐：`RAG-Prompt组装模块设计.md`、`混合检索模块设计.md`  
> 当前实现：`chat_service.py` + `useChatStream.ts` + `markstream-vue`

---

## 1. 模块职责定义

### 为什么流式问答是演示亮点

- 用户感知延迟更低：首 token 即见反馈
- 更接近真实 AI 产品体验（ChatGPT 式打字机）
- 答辩时可完整演示：检索 → Prompt → 流式生成 → 引用卡片

### 为什么选 SSE + fetch，而不是 WebSocket

| SSE + fetch ReadableStream | WebSocket |
|---------------------------|-----------|
| 单向 server→client，契合 LLM 流式输出 | 双向，问答场景多数用不上 |
| 基于 HTTP，走现有 FastAPI / 代理 / 鉴权 | 需独立 WS 路由与心跳 |
| fetch + AbortController 取消简单 | 连接管理更复杂 |
| EventSource 只支持 GET；**fetch 支持 POST** | 适合实时协作、多播 |

**结论：** RAG 问答是「POST 问题 → 流式收答案」，用 `StreamingResponse(text/event-stream)` + `fetch` 最合适。

## 2. 后端流式设计

```
POST /knowledge-bases/{id}/chat/stream
  → 混合检索 → Prompt 组装 → LLM 流式生成
  → StreamingResponse(text/event-stream)
```

### SSE 帧格式（JSON data 行）

```
data: {"type":"start","message_id":"..."}

data: {"type":"token","delta":"你"}

data: {"type":"references","citations":[...]}

data: {"type":"done"}

data: {"type":"error","message":"..."}
```

- **references 单独一帧**，不与正文 token 混合
- 正文结束后、done 之前发送 references

---

## 3. 前端流式消费

1. `fetch(url, { method:'POST', body, signal })`
2. `response.body.getReader()` + `TextDecoder` 累积 buffer
3. 按 `\n\n` 切 SSE 事件，解析 `data:` JSON
4. `token` → 追加 `content`；`references` → 绑定卡片；`done` → 结束 loading
5. `AbortController.abort()` 取消请求

### 粘包 / 半包

- buffer 累积，只处理完整 `\n\n` 块
- JSON 按 type 分发，不完整行留 buffer

---

## 4. 数据协议

| type | 字段 | 说明 |
|------|------|------|
| `start` | message_id | 流开始 |
| `token` | delta | 增量文本 |
| `references` | citations | 引用源列表 |
| `done` | - | 正常结束 |
| `error` | message | 错误信息 |

---

## 5. UI 交互

| 场景 | 做法 |
|------|------|
| 打字机 | `markstream-vue` 增量渲染 Markdown（非 v-html 全量） |
| 回答中 | 助手消息 `status=streaming` + 光标/Spin |
| 出错 | `status=error` + Alert |
| 取消 | 停止按钮 → `abort()` |

## 7. 面试表达

> 我们用 SSE 而不是 WebSocket，因为 RAG 是单向流式输出，POST + fetch 更贴合 HTTP 栈。协议上 token 帧与 references 帧分离，前端用 markstream-vue 增量渲染 Markdown，引用卡片在 references 事件到达后单独展示，避免解析正文里的 JSON。
