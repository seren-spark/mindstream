# 聊天页前端模块设计

> 对齐：`流式问答模块设计.md`、`RAG-Prompt组装模块设计.md`  
> 实现：`frontend/src/views/chat/`、`frontend/src/components/chat/`、`useChatStream.ts`  
> 全站 UI：`frontend/src/assets/styles/ui.css`（动效 token、路由过渡、Arco 柔和化）

---

## 1. 页面职责

| 职责 | 说明 |
|------|------|
| 会话容器 | 选知识库 → 提问 → 看回答，RAG 能力的用户出口 |
| 流式呈现 | SSE `token` / `references` → 打字机 Markdown + 引用卡片 |
| 可信展示 | 引用源可核对，体现「企业知识助手」而非娱乐 IM |
| 异常可控 | loading、停止、失败、空态引导 |

**前端亮点**：POST + fetch 流式协议、增量 Markdown（`markstream-vue`）、单条 assistant 状态机、组件与 composable 分层。

## 3. 消息数据结构

```typescript
type ChatMessageStatus = 'pending' | 'streaming' | 'done' | 'error' | 'cancelled'

interface ChatMessage {
  id: string
  role: 'user' | 'assistant'
  content: string              // user 纯文本；assistant Markdown
  status: ChatMessageStatus
  citations?: CitationRef[]
  errorMessage?: string
}

interface CitationRef {
  index: number
  chunk_id: string
  knowledge_item_id: number
  source_name: string
  display_title: string
  highlight_text: string
  content_excerpt: string
  score: number
  source_location: { page_start?, section_title?, ... }
}
```

**会话级**（页面 / composable）：`messages`、`knowledgeBaseId`、`streaming`（`useChatStream`）、`scrollPinned`（`useChatScroll`）。

## 5. 状态与交互

| 场景 | UI | 逻辑 |
|------|-----|------|
| 首屏空态 | 欢迎文案 + 示例问题 chips | 点击填入输入框 |
| 发送中 | 用户消息立即入列 | `useChatStream.send` |
| 流式中 | Spin → Markdown + 光标；禁用输入 | `status=streaming` |
| 完成 | 展示引用折叠区 | `status=done` 且 `citations` |
| 失败/停止 | Alert | `error` / `cancelled`；可 `cancel()` |
| 滚动 | 贴底自动滚；用户上滑则停 | `useChatScroll` |

```
Empty → Sending → Streaming → Done / Error / Cancelled
```

## 6. Markdown 渲染

| 项 | 方案 |
|----|------|
| 库 | `markstream-vue`（增量 AST，适合流式） |
| 代码高亮 | 库内置 CodeBlock 节点 |
| HTML | **不**用 `v-html` 渲染模型输出 |
| 安全 | 仅 Markdown 组件树；外链 `rel="noopener noreferrer"` |

```vue
<MarkdownRender
  :content="content"
  :typewriter="status === 'streaming'"
  :batch-rendering="true"
  :max-live-nodes="0"
/>
```

## 7. 引用源

- **位置**：助手消息正文下方，`a-collapse` 默认收起。
- **卡片**：`[index]` 标题、`highlight_text` 摘要、页码/章节元信息。
- **映射**：正文 `[n]` 与 `citations[].index` 一致（Prompt 约定）。
- **点击**：跳转知识条目详情（`knowledge_item_id` + `chunk_id` query）。

## 8. SSE 协议（前后端）

```
POST /knowledge-bases/{id}/chat/stream
data: {"type":"start","message_id":"..."}
data: {"type":"token","delta":"..."}
data: {"type":"references","citations":[...]}
data: {"type":"done"}
data: {"type":"error","message":"..."}
```

- `token` 与 `references` **分帧**；引用在流式结束后展示，避免卡片闪烁。

## 9. 面试表达

> 聊天页拆成页面编排、消息 UI、流式 composable、SSE 解析四层。用 POST + fetch ReadableStream 消费 SSE，buffer 按 `\n\n` 防粘包，`AbortController` 停止。Markdown 用 markstream-vue 增量渲染，不用 v-html。引用单独一帧，折叠在消息底部，角标与 `CitationRef.index` 对齐，体现 RAG 可追溯。
