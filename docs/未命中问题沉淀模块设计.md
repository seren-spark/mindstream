# 未命中问题沉淀模块设计

> 对齐：`Dashboard统计模块设计.md`、`流式问答模块设计.md`  
> 闭环：**使用 → 发现缺口 → 沉淀条目 → 命中率提升**

---

## 1. 模块职责

### 价值

用户高频提问但知识库无法回答的问题，是**最有价值的补库信号**。本模块将「未命中」从一次性统计变为**可运营 backlog**。

### AI 产品闭环

```
提问 → RAG 判定未命中 → 写入 unanswered_questions
     → 运营确认/编辑 → 一键转知识条目 → 索引就绪 → 后续提问命中
```

体现：**知识库由使用行为反向驱动完善**，而非静态文档仓库。

---

## 2. 判定策略（Demo 推荐：规则组合）

| 信号 | 条件 | reason |
|------|------|--------|
| 无引用 | `references_json` 为空 | `no_citations` |
| 拒答文案 | 含「无法找到相关信息」 | `reject_phrase` |
| 低分检索 | `top_score < 0.35` 且无引用 | `low_score` |
| 无检索结果 | `hits` 为空 | `no_hits` |

**优先级**：有有效引用 → 不算未命中；否则按上表记录主因。

增强阶段可叠加 LLM 分类，Demo 不做。

---

## 3. 数据模型 `unanswered_questions`

| 字段 | 说明 |
|------|------|
| id | UUID |
| knowledge_base_id | 归属知识库 |
| query_text | 展示用原文 |
| query_norm | 归一化去重键 |
| occurrence_count | 累计次数 |
| first_asked_at / last_asked_at | 时间 |
| status | open / reviewing / resolved / dismissed |
| reason | 未命中原因 |
| top_score | 当次最高检索分 |
| sample_*_message_id | 样本消息 |
| suggested_title / suggested_summary | 转条目草稿 |
| resolved_item_id | 沉淀后的条目 ID |

唯一约束：`(knowledge_base_id, query_norm)`

---

## 4. 页面设计

| 位置 | 作用 |
|------|------|
| Dashboard 表格 | 摘要 Top N +「查看全部」 |
| **独立页 `/gaps`** | 筛选、状态流转、转条目 |

操作：忽略、标记处理中、**一键转知识条目**（弹窗预填标题/摘要）。

---

## 5. API

前缀：`/api/knowledge-bases/{kb_id}/unanswered-questions`

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/` | 列表（status、分页） |
| GET | `/{id}` | 详情 |
| PATCH | `/{id}` | 更新状态/建议文案 |
| POST | `/{id}/dismiss` | 忽略 |
| POST | `/{id}/resolve` | 转知识条目并 resolved |

写入：流式问答完成时 `UnansweredService.record_from_chat()` 自动 upsert。

---

## 6. 面试表达

> 我们把每次问答的检索分、引用列表和拒答规则组合判定未命中，并沉淀到独立表聚合频次。运营在缺口页一键生成知识条目，形成「使用促进沉淀」闭环，让知识库越用越全。
